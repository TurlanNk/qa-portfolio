# Smoke Checklist – Web Application

## Purpose
Quick verification that the main functionality of the application works after a new build or deployment.

---

### 1. Application Launch
- Website opens without errors
- Main page loads correctly

### 2. User Registration
- User can successfully register with valid data

### 3. User Login
- User can log in with valid credentials
- User session is created

### 4. Access Control
- Unauthorized user cannot access protected pages (e.g., /profile)

### 5. Core Business Functionality
- User can create/publish content (e.g., post/advertisement)
- Published content is visible

### 6. Logout
- User can log out successfully
- Session is terminated
